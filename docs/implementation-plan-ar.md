# خطة تنفيذ نظام السوبرماركت (Arabic Roadmap)

## الهدف
تنفيذ نظام سوبرماركت احترافي (منتجات غذائية وألبان) يشمل: متجر عميل، POS، إدارة منتجات، تقارير، مستخدمين وصلاحيات، طلبات أونلاين، مع دعم عربي كامل وOffline وسرعة عالية.

## الحالة الحالية (ملخص سريع)
- يوجد أساس جيد لـ Next.js + Prisma + POS + Storefront.
- يوجد سلة مشتريات وحساب إجمالي وضريبة.
- يوجد APIs أساسية للمنتجات والطلبات والمستخدمين.
- يوجد Offline modules مبدئية + Service Worker.

## Backlog حسب الأولوية

### P0 (حرج - تشغيل البيع الفعلي)
1. POS بحث سريع + باركود مباشر وإضافة للفاتورة.
2. إتمام Checkout حقيقي (حفظ الطلب + خصم مخزون + حركة خزنة).
3. طباعة إيصال من شاشة الكاشير.
4. قواعد الضريبة والخصم بشكل صحيح داخل الطلب.
5. إدارة صلاحيات (كاشير/مدير/أدمن) في الشاشات والـ APIs.

### P1 (أساسي للعمل اليومي)
1. CRUD كامل للمنتجات والتصنيفات.
2. تنبيهات مخزون منخفض حقيقية من قاعدة البيانات.
3. إدارة الطلبات الأونلاين وحالات الطلب.
4. تقارير يومية/شهرية + الأكثر مبيعاً + تقرير خزنة.

### P2 (احترافي وتميّز)
1. Offline-first في POS (بيع بدون انترنت + مزامنة لاحقة).
2. تحسين الأداء (Caching, Query optimization, image optimization).
3. تحسين UX للموبايل والكاشير.
4. نسخ احتياطي واسترجاع بضغطة واحدة.
